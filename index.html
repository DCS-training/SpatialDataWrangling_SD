Spatial Data Wrangling in QGIS

Preparing Data for Statistical Analysis: Edinburgh Case Study

1. Introduction

Many types of sources used by social scientists are inherently spatial where spatial data is treated as a primary variable. We could be interested in how, for instance, tree density is associated with indicators of wellbeing, or environmental injustice where poorer neighbourhoods are associated with worse environmental conditions such as noise.

Lesson Goals

  • Aggregate vector and raster information to polygons to aid in visualising patterns.
  • Construct polygon bins and apply spatial join techniques for vector-based point data.
  • Align raster data with polygon bins using zonal operations.
  • Export aggregated datasets for further statistical analysis.

2. Setup & Requirements

This tutorial builds upon foundational skills in GIS, especially adding layers and managing Coordinate Reference Systems (CRS).

  1. Download the SilentDiscoGeo (.zip) folder.
  2. Relocate the folder to your Desktop for easy access.
  3. Ensure you have the latest version of QGIS installed.

Open ProjectGeo.qgz in QGIS to begin. Add the Tree.shp and Edinburgh_Ward_Boundaries.shp layers from the Vectors folder.

[Image: Initial Tree Distribution Map]

3. Aggregating Vector Point Data

To understand the relationship of trees in Edinburgh, we will create a Choropleth Map showing the number of trees in each Ward Boundary.

Steps:

  1. Go to Processing > Toolbox and search for "Join attributes by location (summary)".
  2. Join to features in: Edinburgh_Ward_Boundaries.
  3. By comparing as: Trees.
  4. Fields to summarise: Check ID.
  5. Summaries to calculate: Check count.
  6. Run the tool and style the resulting layer using Graduated Symbology with the OBJECTID_count value.
[Image: Tree Count Choropleth]

4. Aggregating Raster Noise Data

Now we will load the noise level data (HORE_CONSOLIDATED_LDEN.tif). LDEN records noise levels in decibels (dB).

Zonal Statistics:

To get a representative average noise value for each Ward, we use Zonal Statistics.

  • Input layer: Edinburgh_Ward_Boundaries.
  • Raster layer: HORE_CONSOLIDATED_LDEN.
  • Statistics to calculate: mean.

Style this layer using a red-to-blue gradient to represent high and low noise levels.

[Image: Noise Level Choropleth]

5. Constructing Polygon Bins (Hexagons)

To perform statistical analysis, we need uniform sample "bins".

  1. Go to Vector > Research Tools > Create Grid.
  2. Grid Type: Hexagon.
  3. Extent: Calculate from Edinburgh_Ward_Boundaries.
  4. Spacing: 500 meters.
  5. Use Select by Location to trim the grid to the city boundaries.
[Image: Hexagonal Grid Overlay]

7. Conclusion

A Spearman correlation is a brilliant tool for "spotting" a pattern, but spatial tools are required to "prove" it by accounting for the complex, overlapping layers of a living city.
