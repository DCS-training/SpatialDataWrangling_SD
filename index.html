Spatial Data Wrangling in QGIS

This tutorial forms part of the training in geospatial analysis, building upon foundational GIS skills like layer management and Coordinate Reference Systems (CRS)[cite: 15].

Lesson Goals

  • Aggregate vector and raster information to polygons[cite: 11].
  • Construct polygon bins and apply spatial join techniques[cite: 12].
  • Extract and align raster data using zonal operations[cite: 13].
  • Export aggregated datasets for exploration[cite: 14].

Introduction

Spatial aggregation is the process of grouping individual data points into summarised geographic units to reveal patterns invisible at the individual point level[cite: 20, 21].

Vector Aggregation

To count trees within ward boundaries, we use spatial join techniques.

Key Step: Use the Join attributes by location (summary) tool. Set the Input Layer to your ward boundaries and the Join Layer to your tree points[cite: 37, 42].

Raster Aggregation

Raster data, such as noise levels (LDEN), requires calculating mean values within polygons.

Key Step: Open the Zonal Statistics tool. Select your polygons as the input and the .tif file as your raster to calculate the mean[cite: 73, 74].

Constructing Bins for Aggregation

Hexagonal bins provide a uniform grid for more precise statistical comparison than irregular administrative boundaries[cite: 87, 88].

Key Step: Use Create Grids (Vector > Research Tools) and set the type to Hexagon with 500m spacing[cite: 89, 90].

Exploration of Spatial Relationship in QGIS

We use Spearman’s Rho to determine if tree density correlates with noise levels[cite: 184, 185]. Our findings in Edinburgh suggest a weak positive relationship (0.0587), often due to roadside planting[cite: 187, 196].

Notes

Always check for spatial dependency (autocorrelation). Standard statistics assume each bin is an independent "island," but geographic data often "bleeds" across boundaries[cite: 199, 201].
