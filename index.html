Spatial Data Wrangling in QGIS

Many types of sources used by social scientists are inherently spatial where spatial data is treated as a primary variable[cite: 3]. We could be interested in how for instance, tree density is associated with indicators of wellbeing; environmental injustice where poorer neighbourhoods are associated with worse environmental conditions such as noise[cite: 4, 5]. There are different methods looking at these research questions[cite: 6].

One of the methods is to ‘bin’ disparate spatial datasets into a uniform grids using QGIS[cite: 7]. This process allows you to standardise geographic information, making it possible to compare variables that were originally collected in different formats and to perform further inferential analysis[cite: 8].

This tutorial is going to walk you through the steps to create these ‘bins’ and to perform relevant visualisation and preliminary statistics that hopefully could aid in performing your own research questions[cite: 9]. Here are the lesson goals:

  • To aggregate vector and raster information into polygons to aid in the visualisation of geospatial patterns[cite: 11].
  • To construct polygon bins and apply spatial join techniques, aggregating vector-based point data to create a multi-variable dataset[cite: 12].
  • To extract and align raster data with polygon bins using zonal operations to enhance multi-variable datasets[cite: 13].
  • To export aggregated datasets for further statistical analysis or rapid exploration using QGIS consoles[cite: 14].

This tutorial forms part of the training in geospatial analysis and builds upon foundational skills in GIS, particularly adding layers and managing Coordinate Reference Systems (CRS)[cite: 15].

Before beginning, please download SilentDiscoGeo (.zip)[cite: 16]. This tutorial assumes you are working with this specific set of environmental data relating to the City of Edinburgh and wish to aggregate this information for statistical analysis[cite: 16].

After downloading the ZIP folder, please move it to your desktop for easy access and reference as you follow the tutorial[cite: 17]. You will also need QGIS installed on your computer; please download the latest version from [here][cite: 18].

Introduction

What is spatial aggregation, and why do we aggregate spatial information? [cite: 19]

Spatial aggregation is the process of grouping individual data points—such as tree locations or noise sensor readings—into summarised geographic units[cite: 20]. Unlike standard data grouping (such as sorting sales by month), spatial aggregation utilises locational relationships to reveal patterns, trends, and clusters that are invisible at the individual point level[cite: 21].

To put this into context, let’s open the folder you downloaded[cite: 22]. Go to your desktop and open the unzipped SilentDiscoGeo folder[cite: 23]. Inside, open ProjectGeo.qgz in QGIS (you should be able to open this by simply double-clicking the file if you have QGIS installed)[cite: 24].

Once the project is open, follow these steps to add your layers[cite: 25]:

Key Step: Adding Layers

1. Add the Trees layer:

  • Go to Layer > Add Layer > Add Vector Layer.
  • Click the three dots (browse button) next to the "Source" box.
  • Navigate to the Vectors > Trees folder and select Tree.shp.
  • Click Open, then Add.

2. Add the Edinburgh Ward Boundaries:

  • To provide spatial context, go to Layer > Add Layer > Add Vector Layer.
  • Click the three dots next to the "Source" box.
  • Navigate to the Vectors folder and select Edinburgh_Ward_Boundaries.shp.
  • Click Open, then Add.

What conclusions can you draw regarding the distribution of trees across Edinburgh’s ward boundaries? [cite: 30] You will likely realise immediately that this is quite difficult to determine because the points overlap, making it nearly impossible to visualise patterns across the different boundaries[cite: 31]. Therefore, we must find a more effective way to visualise the distribution of trees in Edinburgh, which is where spatial aggregation becomes essential[cite: 32].

It would be helpful if we could represent the number of trees within each ward boundary as a Choropleth Map, thereby identifying which areas have a higher tree count and which have fewer[cite: 33]. This illustrates the primary reason we aggregate geographical data: it helps us to visualise geospatial relationships better[cite: 34].

Vector Aggregation

To create a choropleth map for the tree data with respect to the Edinburgh Ward Boundaries, we need to firstly calculate how many trees there are for each Ward Boundary[cite: 36]. To sum up how many trees there are in each ward, we will be using a very useful tool in QGIS, which is “Join attributes by location (summary)”[cite: 37].

Key Step: Aggregating Vector Point Data into Polygons
  • To open this tool, you will need to open the Processing Toolbox window[cite: 38].
  • At the top of the menu bar, go to Processing > Toolbox[cite: 39].
  • Inside the window, in the toolbar, type “location”, then choose Join attributes by location (summary)[cite: 41].
  • In the window that pops up, choose Edinburgh_Ward_Boundaries as Join to features in, and choose Trees as By comparing to[cite: 42].
  • For Fields to summarise, click the three dots and check ID[cite: 43].
  • For Summaries to calculate, click the three dots, check count, then OK[cite: 44].
  • Click Run[cite: 45].

A new layer called “Joined layer” will appear[cite: 46]. Now we need to make the layer show the number of trees per ward in a gradient[cite: 47].

Key Step: Visualising the Count
  • Right-click onto Joined layer > Properties[cite: 48].
  • Select Symbology on the left, and choose Graduated from the top dropdown[cite: 49].
  • For Value, select OBJECTID_count[cite: 50].
  • Choose a Color ramp (e.g., shades of green)[cite: 50].
  • Set Mode to Equal Count (Quantile) and Classes to 5[cite: 51].
  • Click Classify, then OK[cite: 52].

Raster Aggregation

Standard spatial joins cannot compute raster data like noise levels (LDEN recorded in dB)[cite: 57]. We need a different approach to create a Choropleth Map using raster data[cite: 56].

Key Step: Load HORE_CONSOLIDATED_LDEN.tif via Layer > Add Layer > Add Raster Layer[cite: 62]. Use the Zonal Statistics tool in the Processing Toolbox[cite: 73]. Set the Input layer to the boundaries and the Raster layer to the noise data, then check mean[cite: 74, 75].

Constructing Bins for Aggregation

To perform statistical inference, we construct uniform sample 'bins'[cite: 87]. Each bin serves as an ID for an individual record in your dataset[cite: 88].

Key Step: Go to Vector > Research Tools > Create Grid[cite: 89]. Select Hexagon and set spacing to 500 metres[cite: 90, 91]. Use Select by Location to trim grids that intersect with the ward boundaries[cite: 96, 97].
