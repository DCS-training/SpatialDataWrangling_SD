Spatial Data Wrangling in QGIS

Many types of sources used by social scientists are inherently spatial, treating spatial data as a primary variable. We might be interested, for instance, in how tree density is associated with indicators of wellbeing, or in environmental injustice, where poorer neighbourhoods are associated with worse environmental conditions, such as noise. Various methods exist to address these research questions. One such method involves ‘binning’ disparate spatial datasets into uniform grids using QGIS. This process allows you to standardise geographic information, making it possible to compare variables originally collected in different formats and to perform further inferential analysis. This tutorial will walk you through the steps to create these ‘bins’ and perform relevant visualisation and preliminary statistics to aid you in exploring your own research questions.

Lesson Goals

  • To aggregate vector and raster information to polygons to aid in visualising patterns of geospatial information[cite: 11].
  • To construct polygon bins and apply spatial join techniques to aggregate vector-based point data[cite: 12].
  • To extract and align raster data with polygon bins using zonal operations[cite: 13].
  • To export aggregated datasets for further statistical analysis[cite: 14].

Introduction

Spatial aggregation is the process of grouping individual data points—such as tree locations or noise sensors—into summarised geographic units[cite: 20]. This reveals patterns, trends, and clusters that are invisible at the individual point level[cite: 21].

Getting Started: Open ProjectGeo.qgz in QGIS[cite: 24]. Add Tree.shp and Edinburgh_Ward_Boundaries.shp via Layer > Add Layer > Add Vector Layer[cite: 26, 27, 28].

Vector Aggregation

To create a choropleth map for tree data, we first calculate the number of trees in each Ward Boundary[cite: 36].

Key Step: Open the Processing Toolbox (Processing > Toolbox) and search for Join attributes by location (summary)[cite: 39, 41].
  • Join to features in: Edinburgh_Ward_Boundaries[cite: 42].
  • By comparing to: Trees[cite: 42].
  • Summaries to calculate: count[cite: 44].

Style the result by right-clicking the layer > Properties > Symbology. Choose Graduated and select OBJECTID_count as the value[cite: 48, 49, 50].

Raster Aggregation

Raster data, like the noise level (LDEN recorded in dB), cannot be processed with the standard Join tool[cite: 57, 61].

Key Step: Load HORE_CONSOLIDATED_LDEN.tif[cite: 59]. Use the Zonal Statistics tool from the Processing Toolbox[cite: 73]. Set the Input layer to the boundaries and the Raster layer to the noise data, then check mean[cite: 74, 75].

Constructing Bins for Aggregation

To perform statistical inference, we must construct uniform sample 'bins'[cite: 87]. Each bin acts as an ID for an individual observation in your analysis[cite: 88].

Key Step: Go to Vector > Research Tools > Create Grid[cite: 89]. Select Hexagon and set spacing to 500 meters[cite: 90, 91]. Use Select by Location to trim the grid to the city boundaries[cite: 96].

Notes

Null values in your data represent missing information[cite: 122]. Use the Field Calculator with coalesce("OBJECTID_count", 0) to fix these[cite: 126, 127]. Always be aware of spatial autocorrelation, which can make relationships look significant when they are a byproduct of urban layout[cite: 199, 202].
